**Before reporting:**

### 4. ğŸ’» Code Contributions

Want to contribute code? Awesome!

**Getting Started:**

```bash
# 1. Fork the repository
# Click "Fork" on GitHub

# 2. Clone your fork
git clone https://github.com/YOUR-USERNAME/sf-pro-toolkit.git
cd sf-pro-toolkit

# 3. Create a feature branch
git checkout -b feature/your-feature-name

# 4. Install dependencies
npm install

# 5. Load the extension in Chrome
# Open chrome://extensions/
# Enable "Developer mode"
# Click "Load unpacked"
# Select the sf-pro-toolkit directory
```

**Development Workflow:**

1. **Make Your Changes**: Edit the relevant files
2. **Test Thoroughly**: Test in multiple scenarios (see [Testing Guide](https://github.com/sidbhat/sap-pro-toolkit/blob/main/TESTING-GUIDE.md))
3. **Follow Code Standards**: See [Coding Standards](https://github.com/sidbhat/sap-pro-toolkit/blob/main/.clinerules/coding.md)
4. **Run Security Checks**: Ensure no sensitive data is committed
5. **Commit Changes**: Use clear, descriptive commit messages
6. **Push to Your Fork**: `git push origin feature/your-feature-name`
7. **Create Pull Request**: Submit PR with detailed description

**Code Areas:**
- `panel/`: Main extension UI and logic
- `content/`: Content scripts for page interaction
- `background/`: Background service worker
- `resources/`: Profile JSON files
- `_locales/`: Translations (10 languages)

## ğŸ”’ Security & Sanitization Guidelines

<Callout type="error">
  **Critical**: Never commit sensitive data to the public repository!
</Callout>

### âŒ What to NEVER Include

- Real production hostnames (e.g., `mycompany.sapsf.com`)
- Company IDs or tenant identifiers (e.g., `SFSALES123456`)
- Credentials, API keys, or tokens
- Internal employee information (names, IDs, emails)
- Proprietary business data or configurations
- Customer-specific information

### âœ… What is Safe to Include

- Generic documentation URLs (SAP Help Portal, Community)
- Public roadmap links
- SAP Note numbers (publicly available)
- Example configurations with placeholders
- Template code snippets
- Generic test user patterns (`test-employee@test.com`)

### ğŸ›¡ï¸ Before Submitting

**Run Security Checks:**

```bash
# Search for credentials
grep -r "password\|credential\|token\|apikey" --include="*.json" resources/

# Check for internal emails
grep -r "@.*\.corp\.sap\|@sap\.com" --include="*.json" resources/

# Look for company IDs
grep -r "SFSALES\|company=" --include="*.json" resources/
```

**Use Placeholders:**
```json
{
  "hostname": "your-company.sapsf.com",
  "notes": "[CUSTOMIZE] Replace with your production hostname"
}
```
